# AI智能体系统 - 数据需求文档

## 一、项目概述

本系统是一个面向Python学习的AI智能体系统，需要存储和管理以下几类数据：
- 用户数据（认证、画像）
- 学习数据（记录、进度）
- 知识数据（知识库、文档）
- 对话数据（聊天历史）

## 二、当前已定义的数据表

### 2.1 UserProfile（用户画像表）

| 字段名 | 数据类型 | 是否必填 | 说明 |
|--------|----------|----------|------|
| user_id | String(50) | ✅ | 用户唯一标识（主键） |
| knowledge_base | String(50) | ✅ | 知识基础：初学者/基础薄弱/中等/良好/优秀 |
| cognitive_style | String(50) | ✅ | 认知风格：视觉型/听觉型/动觉型/读写型/混合型 |
| weak_points | Text | ❌ | 易错点，逗号分隔 |
| interest | String(100) | ❌ | 兴趣领域，逗号分隔 |
| learning_pace | String(20) | ✅ | 学习节奏：快速/中等/慢速/时快时慢 |
| emotional_state | String(20) | ✅ | 情绪状态：积极/中性/焦虑/疲惫/抵触 |
| created_at | DateTime | ✅ | 创建时间 |
| updated_at | DateTime | ✅ | 更新时间 |

**用途：** 用于个性化学习资料生成、展示用户学习特征

## 四、各角色需要的数据汇总

### 4.1 角色分工表

| 数据类型 | AI负责人（你） | 前端开发（B） | 后端开发（C） |
|----------|---------------|---------------|---------------|
| **用户认证** | ❌ | ✅ 登录/注册界面 | ✅ 实现认证逻辑 |
| **用户画像** | ✅ 读取用于个性化生成 | ✅ 展示/编辑画像 | ✅ CRUD操作 |
| **学习记录** | ✅ 读取历史数据 | ✅ 展示学习历史 | ✅ CRUD操作 |
| **知识库** | ✅ 读取用于RAG检索 | ✅ 搜索/浏览知识库 | ✅ CRUD操作 |
| **对话历史** | ✅ 读取上下文 | ✅ 展示聊天记录 | ✅ 存储对话 |
| **学习进度** | ✅ 分析进度 | ✅ 展示进度条 | ✅ 记录进度 |

### 4.2 数据流转关系

```
用户注册/登录 → User表
    ↓
用户对话 → ChatHistory表
    ↓
AI分析画像 → UserProfile表（更新/创建）
    ↓
RAG检索知识 → KnowledgeBase表（读取）+ FAISS向量库（检索）
    ↓
生成学习资料 → LearningRecord表（存储）
    ↓
制定学习路线 → LearningProgress表（初始化）
    ↓
用户学习 → LearningProgress表（更新进度）
```

## 五、Python学习知识库数据需求

### 5.1 知识库来源方式

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **本地文档导入** | 下载Python学习PPT/PDF，放入data/目录 | 数据质量高、稳定、无需网络 | 需要手动更新、数据量有限 |
| **数据库存储** | 把知识存入关系数据库，再同步到RAG | 结构化管理、便于增删改查 | 需要开发同步逻辑 |
| **混合模式** | 本地文档作为基础库 + 数据库动态补充 | 兼顾稳定性和灵活性 | 实现稍复杂 |

### 5.2 建议收集的Python学习资料

**基础入门类：**
- Python编程：从入门到实践（PDF/TXT）
- Python官方教程（MD/TXT）
- Python基础语法笔记（MD）
- Python环境搭建指南（MD）

**核心语法类：**
- 变量与数据类型（TXT/MD）
- 控制流程（if/for/while）（TXT/MD）
- 函数定义与调用（TXT/MD）
- 面向对象编程（TXT/MD）
- 异常处理（TXT/MD）

**常用库类：**
- NumPy教程（PDF/MD）
- Pandas数据分析（PDF/MD）
- Matplotlib可视化（PDF/MD）
- Requests网络请求（MD）
- Flask/Django Web开发（PDF/MD）

**进阶内容类：**
- 数据结构与算法（PDF/MD）
- 并发编程（TXT/MD）
- 设计模式（TXT/MD）
- 机器学习入门（PDF/MD）

### 5.3 文档格式要求

| 格式 | 支持情况 | 说明 |
|------|----------|------|
| .txt | ✅ 支持 | 纯文本文件，简单直接 |
| .md | ✅ 支持 | Markdown格式，支持排版 |
| .pdf | ✅ 支持 | 需要安装PyPDF2库 |
| .ppt | ❌ 暂不支持 | 建议转换为PDF或提取文本 |
| .docx | ❌ 暂不支持 | 建议转换为TXT或MD |

### 5.4 数据质量要求

1. **准确性**：内容准确无误，无错误信息
2. **完整性**：每个知识点覆盖完整，无遗漏
3. **结构化**：便于切分成500字符左右的文档块
4. **标注清晰**：包含分类、难度等级等元数据
5. **格式规范**：统一编码为UTF-8

## 七、数据安全与隐私

### 7.1 数据加密
- 密码使用bcrypt或scrypt加密存储
- JWT密钥定期轮换
- 敏感数据传输使用HTTPS

### 7.2 访问控制
- 基于角色的权限控制（RBAC）
- 用户只能访问自己的数据
- 管理员可查看所有数据

### 7.3 数据备份
- 定期备份数据库
- 备份文件加密存储
- 测试环境使用脱敏数据
