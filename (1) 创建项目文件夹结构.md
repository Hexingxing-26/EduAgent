## 二、API接口文档（给前端团队）

您的模块提供以下接口：

| 接口             | 方法   | 说明            |
| -------------- | ---- | ------------- |
| `/`            | GET  | 健康检查          |
| `/chat/stream` | POST | **核心接口：流式对话** |
| `/docs`        | GET  | Swagger文档     |

### `/chat/stream` 接口详情

**请求体**：

```json
{
  "user_id": "string (用户唯一标识)",
  "message": "string (用户提问内容)"
}
```

**响应格式**（Server-Sent Events 流式返回）：

```json
// 画像分析完成
{"type": "画像", "content": {"knowledge_base": "初学者", "cognitive_style": "视觉型", ...}}

// 学习资料生成完成
{"type": "学习资料", "content": {"type": "multi_resource", "content": "..."}}

// 审核状态
{"type": "状态", "content": "审核通过" | "审核失败重试"}

// 学习路线生成完成
{"type": "学习路线", "content": [{"step_name": "...", "action": "...", "time_minutes": 20}]}

// 全部完成
{"type": "完成", "content": "全部生成完毕"}
```

### 前端接入示例

```javascript
const eventSource = new EventSource('/chat/stream', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({user_id: 'xxx', message: '我想学习决策树'})
});

eventSource.addEventListener('message', (event) => {
  const data = JSON.parse(event.data.replace('data: ', ''));
  switch(data.type) {
    case '画像': // 更新画像展示
    case '学习资料': // 渲染Markdown内容
    case '状态': // 显示审核状态
    case '学习路线': // 展示学习步骤
    case '完成': // 结束对话
  }
});
```

## 四、与数据库团队协作

### 当前数据需求

| 数据类型    | 当前实现            | 未来需数据库支持  |
| ------- | --------------- | --------- |
| 用户画像    | 内存状态            | 用户画像持久化存储 |
| 学习记录    | 无               | 学习进度追踪    |
| 知识库     | `data/demo.txt` | 向量数据库存储   |
| 生成的学习资料 | 无               | 资料版本管理    |

### 数据库表设计建议

```sql
-- 用户画像表
CREATE TABLE user_profiles (
    user_id VARCHAR(50) PRIMARY KEY,
    knowledge_base VARCHAR(50),
    cognitive_style VARCHAR(50),
    weak_points TEXT,
    interest VARCHAR(100),
    learning_pace VARCHAR(20),
    emotional_state VARCHAR(20),
    updated_at TIMESTAMP
);

-- 学习记录
CREATE TABLE learning_records (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(50),
    topic VARCHAR(200),
    generated_resource TEXT,
    learning_path JSONB,
    created_at TIMESTAMP
);
```

## 五、Docker部署方案

创建 `Dockerfile` 和 `docker-compose.yml`：

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  agent-service:
    build: ./AI_Agent_System
    ports:
      - "8000:8000"
    environment:
      - SPARK_API_KEY=${SPARK_API_KEY}
      - SPARK_API_SECRET=${SPARK_API_SECRET}
      - SPARK_MODEL=spark-x2
```
