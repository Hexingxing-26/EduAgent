《Python基础教程（第3版）》· 教师版教学大纲

课程性质：程序设计语言基础课 / Python入门课程
适用对象：本科一年级、高职高专、零基础转专业学生
先修要求：无（或具备基本的计算机操作能力）
总建议学时：48～64学时（理论32～40 + 实验/实践16～24）

教学总体目标：
1. 掌握Python语言的核心语法与编程范式（面向过程、面向对象、函数式）。
2. 能够熟练使用Python标准库完成常见的文件、网络、数据处理任务。
3. 建立良好的编程习惯：代码规范、测试意识、模块化设计。
4. 具备独立完成中小型项目（300～800行代码）的能力。

第1章 快速上手：基础知识（2～3学时理论 + 1学时实验）
--------------------------------------------------------
【教学目标】
- 了解Python的运行方式（交互式、脚本式）
- 掌握变量、表达式、语句的基本概念
- 掌握常用的内置函数和模块导入方式
- 理解字符串的基础表示（含Unicode、bytes的概念）

【教学重点】
- 变量的赋值与命名规则
- 数值运算（`/` 与 `//`，`%`，`**`）
- `input()` 与 `print()` 的基本使用
- 模块导入（`import` 与 `from ... import`）
- 原始字符串（`r''`）与长字符串（三引号）

【教学难点】
- 整数除法向下取整（尤其是负数情况）
- `str` 与 `repr` 的区别
- 编码/解码（Unicode vs bytes）的初步理解（可降低要求）

【建议教学方式】
- 以“交互式解释器即计算器”开场，边讲边练
- 第1次实验课：完成一个温度转换或BMI计算器

【作业/测验建议】
- 课后作业：编写程序，接收用户输入的年月日，输出对应的日期格式（参考代码清单2-1）
- 小测验：`10 // -3`、`10 % -3` 的结果；`r'C:\now'` 的输出

【教学重点】

- 切片赋值（替换、插入、删除）

- 列表推导的引入（作为后续章节的铺垫）

【建议教学方式】
- 对比教学：列表 vs 元组，`sort()` vs `sorted()`
- 实验：实现一个简单的名片管理器（增、删、查、列）

【作业/测验建议】
- 编程题：给定列表 `[1,2,3,4,5]`，分别用切片和列表方法完成元素的插入、删除、反转


第3章 字符串（2学时理论 + 1学时实验）
----------------------------------------
【教学目标】

- 了解字符串判断方法（`isdigit`、`isalpha` 等）

【教学重点】

- 字符串的 `split` 与 `join` 配对使用
- `strip` 在数据清洗中的作用

【教学难点】
- 格式说明符：对齐（`<`, `>`, `^`）、填充、精度、千位分隔符
- `translate` 与 `maketrans` 的使用（可作选讲）

【建议教学方式】
- 从旧式 `%` 格式化引入，过渡到新式 `format()`，说明为什么推荐新式
- 实验：编写一个日志格式化器，将 `[时间] [级别] 消息` 格式化为表格输出

【作业/测验建议】
- 编程题：将用户输入的字符串中的数字全部提取出来，并组成新字符串
- 课堂练习：`" hello ".strip()`、`"1,2,3".split(",")`、`"-".join(["a","b"])`

第6章 抽象（函数）（3学时理论 + 2学时实验）
-----------------------------------------------
【教学目标】
- 理解函数的定义与调用
- 掌握参数传递机制（位置参数、关键字参数、默认值）
- 理解局部变量与全局变量的作用域
- 初步了解递归

【教学重点】
- `def` 与 `return`
- 参数类型：位置参数、关键字参数、默认参数
- 可变参数：`*args` 与 `**kwargs`
- 作用域：局部 vs 全局，`global` 的用法

【教学难点】
- 参数的传递：不可变对象 vs 可变对象（修改 vs 重新赋值）
- 默认参数为可变对象时的陷阱（如 `def f(lst=[])`）
- 递归的思维模型（基线条件 + 递归条件）

【建议教学方式】
- 先从数学函数（如 `def square(x):`）入手
- 通过“交换变量”演示序列解包
- 递归教学可采用“阶乘”“斐波那契数列”两个经典案例
- 实验：实现一个函数，接受任意多个数字，返回它们的和与平均值

【作业/测验建议】
- 编程题：实现一个递归函数 `power(x, n)` 计算 x 的 n 次方
- 简答题：`def func(lst=[])` 为什么是不安全的写法？

【建议教学方式】
